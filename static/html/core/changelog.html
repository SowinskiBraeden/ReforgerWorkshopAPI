Changelog

Updates for the Reforger Mods API, mod browser, config tools, caching behavior, and documentation.

v1.3.0

Subscription Billing and API Keys

Released 2026-07-12
Latest

Added

  • Stripe Checkout support for paid API subscriptions.
  • Developer and Pro API plans with server-side mapped Stripe price IDs.
  • API key generation, one-time reveal, hash-only storage, and validation.
  • Subscription-aware API access and rate-limit tiers.
  • Stripe webhook handling for checkout, subscription, and invoice lifecycle events.
  • Customer Portal support for billing management.

Security

  • API keys are shown only once and stored as hashes.
  • Stripe webhooks are verified using signing secrets.
  • Stripe secrets are configured through environment variables only.

No existing public API response schemas were changed.

v1.2.0

Server Admin Tools and Persistent Indexing

Released 2026-07-10

Added

  • Public Arma Reforger mod browser at /arma-reforger-mods/, powered by the public API, with search, sorting, pagination, mod detail pages, copy-mod-ID and copy-JSON actions, and correct handling of 202 Accepted refreshes.
  • Config.json tools for server admins: a config validator with local, in-browser JSON and structure checks, a config generator with a form editor, live JSON preview, import/export, and a mod manager for the game.mods array with name resolution, reordering, duplicate detection, and dependency suggestions.
  • Shared working config across tools. Add to Config in the mod browser, the mod manager, and the config generator all edit the same browser-only draft.
  • API quickstart page and new guides for config.json, adding mods, troubleshooting, running server mods, API integration, and refresh jobs.
  • API documentation now includes client identity guidance, conditional request examples, and complete code examples in curl, JavaScript, and Go.
  • SEO metadata for all new pages, including canonical URLs, Open Graph tags, structured data, and sitemap entries.
  • Inline Workshop mod search inside the config generator and mod manager.
  • JSON syntax highlighting in the config validator editor and generator live preview.
  • Persistent local indexing for Workshop list pages, mod metadata, mod details, and API response cache entries.
  • SQLite WAL storage under /var/lib/reforgermods-api when INDEX_ENABLED=true.
  • Background refresh jobs for common popular/newest Workshop list pages and bounded mod detail hydration.
  • Canonical cache keys for list, search, sort, page, tag, and mod ID lookups.
  • Internal metrics for persistent cache hits, stale hits, misses, local search, index refreshes, background jobs, and database errors.

Changed

  • Homepage now presents both the API and the server-admin tools.
  • Site navigation includes the tools and guides sections.
  • Tool pages use a full-width desktop layout with refreshed colors and spacing.
  • Common cached list pages and mod details can be served after service restart when persistent indexing is enabled.
  • Background indexing uses low-priority refresh jobs so direct API misses are not blocked behind crawler work.

No API endpoint, response-schema, or integration changes. The deprecated unversioned API aliases remain untouched.

v1.1.0

Resilient Refreshes

Released 2026-07-07

Improved reliability when Arma Reforger Workshop pages are slow or temporarily unavailable.

Changed

  • Cached API responses now return immediately where possible.
  • Stale cached data can be served while the API refreshes it in the background.
  • Uncached requests may return 202 Accepted while fresh Workshop data is being prepared.
  • Added refresh-job status URLs for integrations that want to show loading progress.
  • Added clearer cache, retry, and rate-limit response headers.

Integration Note

Clients should continue handling normal 200 OK responses as before. When an uncached resource returns 202 Accepted, wait for Retry-After and retry the original URL. Polling the Location job URL is optional.

Fixed

  • API requests no longer wait for a full upstream Workshop scrape before responding.
v1.0.0

Release

Released 2026-07-04

v1.0.0 marks the public return of reforgermods.net and the Reforger Mods API after a period offline.

Earlier iterations of the project are retired and are not compatible with this release. This version establishes the current API foundation, deployment model, and public documentation.

Included

  • Read-only versioned /v1 API routes for health checks, mod lists, search, and mod details.
  • Cached metadata sourced from publicly accessible Arma Reforger Workshop pages.
  • Anonymous public rate limiting with proxy-safe client IP handling.
  • Stale-cache fallback for safe read endpoints when upstream Workshop pages are slow or temporarily unavailable.
  • Controlled JSON error envelopes with request IDs for support and debugging.
  • Docker, reverse-proxy, and deployment documentation.
  • Public documentation for API routes, caching behavior, limits, and supported usage.

Notes

  • This is an independent, unofficial project and is not affiliated with or endorsed by Bohemia Interactive.
  • Workshop data is cached and may not always reflect the latest upstream changes immediately.
  • New integrations should use the versioned /v1 routes.
